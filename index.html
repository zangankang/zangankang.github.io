CUDA STUDY
2026/1/9

手写 CUDA Kernel 或 Triton Kernel 来实现自定义的量化算子（特别是反量化+矩阵乘法的融合算子）。

从零基础到能够编写类似 TinyChat 核心 Kernel 的学习路线以及相关的 GitHub 高星项目。

第一阶段：基础夯实（CUDA 编程入门）

在直接写量化算子之前，必须理解 GPU 的硬件架构（SM, Warp, Memory Hierarchy）。

  1. 核心知识点：
    • GPU 架构：Thread Block, Grid, Shared Memory（共享内存）, Register（寄存器）, Global Memory Coalescing（合并访问）。
    • CUDA C++：内存分配 (cudaMalloc)，数据传输，核函数编写。
    • Nsight Systems/Compute：学会用性能分析工具看 Profiling，知道慢在哪里。
  2. 推荐学习资源：
    • 书籍：《Programming Massively Parallel Processors》（PMPP，Hwu & Kirk 著）。这是公认的 CUDA 圣经。
    • 官方文档：NVIDIA CUDA C++ Programming Guide（必读前几章）。
  3. GitHub 练手项目：
    • kluo/cuda_learning: 包含了很多基础的 CUDA 示例代码，适合入门。

第二阶段：矩阵乘法优化（GEMM Optimization）

LLM 的核心计算 99% 是矩阵乘法（GEMM）。这是所有高性能算子开发者的必经之路。 不懂如何优化 FP32 的 GEMM，就无法写出高效的 INT4 GEMM。

  1. 核心挑战：
    • 从 Naive 实现到 Tiled 实现（利用 Shared Memory）。
    • 解决 Bank Conflict（存储体冲突）。
    • Double Buffering（双缓冲）和 Pipeline（流水线）以此掩盖访存延迟。
  2. 必看 GitHub 教程（高星推荐）：
    • 🌟 siboehm/sgemm-optimization (强烈推荐)
      • 描述：这个仓库手把手教你如何一步步优化 SGEMM（单精度矩阵乘），从最慢的版本优化到接近 CuBLAS 的性能。
      • 学习方式：按照它的 Readme，复现每一个 Kernel 版本（Naive -> Tiling -> Vectorize -> Autotuning）。

第三阶段：量化算子开发（Quantization Kernels）

这一阶段正是 AWQ 和 TinyChat 所做的工作：处理低比特数据（INT4/INT8）和内存排布。

  1. 核心知识点：
    • Bit Packing/Unpacking：如何将两个 INT4 数塞进一个 INT8，或者将 8 个 INT4 塞进一个 INT32。
    • Dequantization on-the-fly：正如 AWQ 论文所述，TinyChat 为了利用 SIMD 指令，在计算前瞬间将 INT4 解包并转换为 FP16 进行计算，或者直接调用 INT8 Tensor Core 指令 。
    • Weight Packing：了解为了适应 GPU 访存，权重数据需要经过特殊的重排（Reordering）。
  2. 实战 GitHub 项目：
    • 🌟 mit-han-lab/llm-awq (你上传论文的官方库)
      • 学习重点：深入阅读 awq/kernels 目录下的 CUDA 代码。重点看 csrc/quantization 里的 .cu 文件，理解它是如何写 unpack 函数的。
    • 🌟 GPTQ-for-LLaMA (虽然较老，但代码结构清晰)
      • 学习重点：查看其 quant_cuda_kernel.cu，特别是如何处理 INT4 的解包和矩阵乘。
    • bitsandbytes
      • 描述：这是目前 HuggingFace 默认集成的量化库。
      • 学习重点：学习它如何通过 CUDA 封装 C 接口供 Python 调用，以及 8-bit 优化技巧。

第四阶段：进阶与现代工具（Triton & CUTLASS）

写原生 CUDA（Raw CUDA）虽然性能高但开发极慢。现在 OpenAI 的 Triton 语言正变得非常流行，它能用类似 Python 的语法写出高性能 Kernel。

  1. 学习路线：
    • OpenAI Triton：学习如何用 Triton 写 Fused Attention 或 Quantized Matmul。
    • CUTLASS (NVIDIA)：这是 C++ 模板库，是编写高性能 GEMM 的“工业标准”，但门槛极高。
  2. 推荐 GitHub 资源：
    • 🌟 openai/triton: 官方仓库的 tutorials 文件夹里有非常好的 03-matrix-multiplication.py 教程。
    • Unsloth: 目前最火的微调加速库，他们大量使用 Triton 手写了反向传播 Kernel，非常值得参考。

总结：你的 GitHub 搜索指南

如果想在 GitHub 上自己挖掘资源，可以使用以下关键词组合进行搜索：

  1. 找学习路线：
    • query: "cuda optimization tutorial"
    • query: "learn cuda programming"
  2. 找量化 Kernel 实现：
    • query: "int4 matmul cuda"
    • query: "quantization kernel llm"
    • query: "fp16 int4 gemm"
  3. 找特定算法的 Kernel：
    • query: "awq kernel implementation"
    • query: "w4a16 kernel" (AWQ 论文中提到的 W4A16 模式 )

第一步

从 siboehm/sgemm-optimization 开始。 不要急着写 INT4，先把能在 CUDA 里写出高性能的 FP32 矩阵乘法作为第一个里程碑。一旦理解了 Tiling 和 Memory Coalescing，AWQ 里的 INT4 解包只不过是在读取内存时多加了几行位运算代码而已。
